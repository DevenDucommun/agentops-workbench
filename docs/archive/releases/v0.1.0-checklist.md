# v0.1.0 Release Checklist

Target: first public-ready post-hoc CLI release.

## Release Gate

- [x] `main` CI is green.
- [x] GitGuardian has no open findings.
- [x] `./bin/agentops scan-publication` passes.
- [x] No local database, transcript, `.agentops/`, `.agents/`, or `.env*` files are tracked.
- [x] Fixtures are synthetic or explicitly approved redacted examples.
- [x] Branch protection is enabled, or the GitHub limitation is documented while the repo remains private.

## Product Smoke Test

```bash
bun install --frozen-lockfile
bun run ci
./bin/agentops ingest ./fixtures/sample-session.jsonl
./bin/agentops report --session latest > /tmp/agentops-report.md
./bin/agentops repo-report --session latest > /tmp/agentops-repo-report.md
./bin/agentops repo-report --session latest --format github > /tmp/agentops-pr-comment.md
./bin/agentops scan-publication
test -s /tmp/agentops-report.md
test -s /tmp/agentops-repo-report.md
test -s /tmp/agentops-pr-comment.md
```

## v0.1 Scope

- [x] Post-hoc ingestion for canonical AgentOps JSONL.
- [x] Post-hoc ingestion for PAI export JSONL.
- [x] SQLite-backed local session store.
- [x] Markdown session report.
- [x] Markdown repo report.
- [x] Compact GitHub PR comment report output.
- [x] Configurable risk and evidence checks.
- [x] Public-readiness scan available through the CLI.

## Known Limitations

- Direct Claude/Codex runtime integrations are planned, not implemented.
- Dashboard is planned, not implemented.
- Cost and token reporting depends on the ingested source data.
- GitHub PR comments are generated locally and written to stdout; the CLI does not post comments.
- The public-readiness scan is a baseline guardrail, not a replacement for dedicated secret scanning.
